# Чеклист

Рекомендации которых необходимо придерживаться при разработке.

1. Код должен быть отформатирован в соответствии с правилами eslint.
Для проверки стиля кода можно выполнить команду 
```bash
npm run lint
```
Для применения исправлений команда
```bash
npm run lint-fix
```

2. Для вставки ссылок должен использоваться компонент nuxt-link, чтобы избегать полной перезагрузки страницы при навигации по внутренним страницам.
Пример:
```vue
<nuxt-link to="/catalog/">Текст ссылки</nuxt-link>
```

3. Для вставки инпутов и прочих элементов форм необходимо использовать компоненты из components/Forms для приведения форм в единый стиль. Примеры использования на странице styleguide
4. Для svg иконок необходимо создавать компонент components/Icons по аналогии с тем, что есть. После создания компонента необходимо добавить иконку в styleguide (components/StyleGuide/IconsExample.vue)
5. Страницы с большим количеством кода необходимо логически разбивать на компоненты. Это упрощает дальнейшую поддержку.
